# 인터뷰 가이드

Phase 2에서 참조하는 질문 목록. **한 번에 1~2가지만** 물어본다.

---

## 탐색 순서 & 질문 예시

### 1. 핵심 기능 파악

> "어떤 기능이 가장 중요해요? 사용자가 이 서비스에 들어와서 제일 먼저 하는 게 뭐예요?"

- 모호한 답변이 오면 보기를 제시: "예를 들어 '노래 추천받기' 인가요, '내 취향 저장하기' 인가요?"
- 기능이 너무 많으면: "그 중에서 딱 하나만 골라야 한다면?"

### 2. 타겟 사용자

> "이 걸 누가 쓸 것 같아요? 나이대나 어떤 상황에서 쓸지 떠오르는 게 있나요?"

- 예시: "20대 직장인이 퇴근길에", "음악 좋아하는 친구들끼리"
- 불특정 다수라고 하면: "FLO 앱 사용자를 대상으로 하는 건가요?"

### 3. 사용 환경

> "이건 스마트폰에서 많이 쓸 것 같나요, 아니면 컴퓨터 웹에서도 쓸 건가요?"

- FLO 앱 안에서 열리는 거면 WebView 환경임 → 자동으로 파악

### 4. 데이터 저장 여부

> "혹시 사용자가 뭔가 기록이 남아야 하는 게 있나요? 예를 들어 점수나 닉네임 같은 거요."

**저장이 필요한 신호:**
- 랭킹/순위, 점수, 퀴즈 결과
- 닉네임/프로필
- 댓글/투표/좋아요
- "다시 봤을 때 남아있어야 해"

**저장 불필요 신호:** 순수 콘텐츠 소비, 일회성 이벤트 페이지

### 5. 외부 연동

> "FLO 로그인한 사람만 쓸 수 있어야 하나요, 아니면 누구나 들어올 수 있나요?"

- FLO 로그인 연동 → WebView Provider + Token 인증 (기본 제공됨)
- 카카오/구글 소셜 로그인 → 추가 구현 필요
- 결제 → 추가 논의 필요

### 6. 스코프 아웃

> "이번엔 안 만들어도 되는 건 뭔가요? '나중에 하면 되는 것' 같은 거요."

- 예: "공유 기능은 나중에", "관리자 페이지는 이번엔 없어도 됨"

---

## 인터뷰 판단 기준

| 상황 | 처리 |
|------|------|
| FLO 앱 WebView에서 열림 | WebView Provider 활성화 필요 |
| 외부 이미지/폰트 추가 | CSP 업데이트 필요 |
| 소셜 로그인 (카카오 등) | 별도 OAuth 구현 필요 → 난이도 높음, 사전 안내 |

## 흐름 예시

```
사용자: 음악 퀴즈 게임 만들고 싶어요
AI: 오 재밌겠는데요! 퀴즈를 맞추면 점수가 쌓이는 형식인가요,
    아니면 그냥 정답 맞추는 게임인가요?

사용자: 점수 쌓이고 랭킹도 보고 싶어요
AI: 멋지네요! 그러면 점수와 닉네임이 저장돼야 하겠네요.

    혹시 이건 FLO 앱 안에서 열리는 건가요, 아니면 웹사이트로 따로 만드는 건가요?
```
