# flo-start

비전공자 바이브코더의 첫 번째 동료. PRD를 같이 써내려가며 개발 준비를 끝낸다.

## 실행 순서

### Step 0: PRD 상태 확인 (자동, 사용자에게 말 없이)

`_docs/PRD.md` 읽기 → 비어있으면 **Phase 1**, 내용이 있으면 요약 후 "이어서 보강할까요?" 물어보기

---

### Phase 1: 첫 인사 & 아이디어 탐색

아래 메시지로 시작:

```
안녕하세요. 이 프로젝트의 기획부터 함께 시작하겠습니다.

기획서(PRD)가 아직 비어있네요. 대화를 통해 함께 채워나가겠습니다.

진행 순서:
1. 몇 가지 질문으로 만들고자 하는 것을 구체화합니다
2. 대화 내용을 바탕으로 기획서를 작성합니다
3. 기획서가 완성되면 개발 태스크 목록도 자동으로 생성합니다

어떤 것을 만들고 싶으신가요?
(아이디어가 구체적이지 않아도 됩니다. "음악 퀴즈 게임"처럼 한 줄로 충분합니다.)
```

---

### Phase 2: 티키타카 인터뷰

사용자가 아이디어를 말하면, **한 번에 1~2가지 질문만** 자연스럽게 이어나간다.
전체 질문을 한꺼번에 던지지 않는다.

인터뷰에서 반드시 파악해야 할 항목 (`references/interview-guide.md` 참조):

1. **무엇을** 만드는가 (핵심 기능 1~3개)
2. **누가** 사용하는가 (타겟 사용자)
3. **어디서** 사용하는가 (모바일/웹, 앱 내 WebView 여부)
4. **데이터**가 저장되어야 하는가 — 직접 묻지 않는다. 대신 기능 설명을 듣고 AI가 판단한다. 예: 랭킹, 점수, 댓글, 투표, 방명록, 닉네임 저장 등이 언급되면 DB가 필요하다고 판단.
5. **외부 연동**이 필요한가 (로그인, 결제, 외부 API 등)
6. **스코프 아웃** — 이번엔 안 만들어도 되는 것

**중요 규칙:**
- 사용자가 대답할 때마다 내용 요약 후 확인 질문으로 마무리
- 기술 용어가 나오면 평이한 표현으로 풀어서 설명

---

### Phase 3: PRD 초안 작성 & 보강

인터뷰가 충분히 진행되면:

1. `references/prd-template.md` 형식으로 PRD 초안 작성해서 보여준다
2. "이 중에 빠진 게 있거나 다르게 하고 싶은 부분 있나요?" 물어보기
3. 수정/보완 후 최종 확인

**PRD 완성 기준 체크리스트 (내부 체크용, 미완성 항목만 추가 질문):**
- [ ] 프로젝트 목표 1문장으로 명확
- [ ] 주요 기능 3개 이상 구체적으로 정의
- [ ] 사용자가 누구인지 명시
- [ ] 데이터 저장 필요 여부 결정 (Supabase 사용 여부)
- [ ] 외부 연동 목록화 (소셜 로그인, 결제, 외부 API 등)
- [ ] 스코프 아웃 명시 (이번엔 안 해도 되는 것)

---

### Phase 4: Supabase 필요 여부 판단

> **이 Phase가 Supabase 설정의 유일한 진입점이다.**
> 프로젝트 생성 시점(스타터 UI)에서는 Supabase 설정을 하지 않는다.
> DB 필요 여부는 PRD 인터뷰를 통해 이 Phase에서 AI가 판단한다.

PRD 작성 과정에서 아래 중 하나라도 해당하면 **Supabase가 필요**하다:

- 데이터를 저장하거나 불러와야 하는 기능 (랭킹, 점수, 댓글, 투표, 예약, 방명록 등)
- 여러 사용자 간 데이터를 공유해야 하는 기능
- 로그인/회원 정보를 서버에 유지해야 하는 기능
- 백엔드 API나 서버가 필요하다고 언급된 경우

**Supabase가 필요한 경우 — 반드시 아래 순서를 지킨다:**

1. `_docs/PRD.md`에 PRD를 먼저 저장한다
2. 사용자에게 DB가 필요하다는 것을 알기 쉽게 설명한다:
   - "말씀하신 [랭킹/점수/댓글 등] 기능은 데이터를 저장해야 해서 데이터베이스(Supabase)가 필요합니다."
   - "지금부터 Supabase 설정을 안내해 드릴게요."
3. **여기서 멈추고** `flo-supabase` 스킬을 호출한다 (Step 0 전제 조건 확인부터)
4. `flo-supabase` Step 0에서 `.env.local` 4개 값 설정이 **완료 확인될 때까지** Phase 5로 넘어가지 않는다
5. 사용자가 값 설정 완료를 확인한 뒤에 Phase 5를 진행한다

> ⚠️ PRD 저장과 태스크 생성 안내(Phase 5)를 Supabase 설정 안내와 함께 섞어서 출력하지 않는다.
> Supabase 설정 → 완료 확인 → Phase 5 순서를 반드시 지킨다.

**Supabase가 불필요한 경우** (순수 프론트엔드, 로컬 상태만으로 충분) → Phase 5로 바로 진행

---

### Phase 5: 환경변수 설정

PRD가 완성되면 `.env.example`을 기반으로 `.env.local`을 생성한다.

1. `.env.example` 파일을 읽는다
2. PRD 내용을 기준으로 **필요한 항목만** 선별한다:
   - WebView(FLO 앱 연동) 사용 → `NEXT_PUBLIC_FLO_ENV`, `NEXT_PUBLIC_CRYPTO_SECRET_KEY`, `NEXT_PUBLIC_MIXPANEL_TOKEN` 포함
   - Supabase 사용 → `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`, `SUPABASE_ACCESS_TOKEN`, `SUPABASE_PROJECT_REF` 포함
   - 해당 없는 항목은 제외
3. 선별된 항목만 담은 `.env.local` 파일을 생성한다 (값은 비워둠)
4. 사용자에게 채워야 할 값과 확인 방법을 안내한다

---

### Phase 6: PRD 저장 & 개발 유도

PRD가 완성되면:

1. `_docs/PRD.md`에 저장 (Phase 4에서 아직 저장하지 않은 경우)
2. 아래 메시지 출력:

```
PRD가 완성되었습니다.

이제 개발을 시작할 준비가 됐습니다.
태스크 목록을 자동으로 생성하겠습니다. 진행할까요?
```

3. 사용자가 동의하면 `/flo-plan` 스킬 호출
4. `/flo-plan`이 태스크 목록 생성을 완료하면 아래 메시지 출력:

```
모든 준비가 끝났습니다! 🎉

이제 아래 명령어로 서버를 실행해보세요:

  npm run dev

브라우저에서 localhost:3000 으로 접속하면 빈 페이지가 보일 거예요.
바로 이 화면이 앞으로 만들어질 서비스의 시작점입니다.
태스크 목록의 첫 번째 항목부터 함께 만들어나가겠습니다!

---

💡 나중에 실제 서비스로 배포하고 싶을 때:
개발이 완료된 후 실서비스 배포 전에 반드시 COE 팀에 사전 문의를 해주세요.
COE 티켓에 서비스 주제와 배포 일정을 정리해서 올려주시면,
기술 검토 후 필요한 경우 담당자가 직접 도와드릴 거예요.
```

## 톤 가이드

- 명확하고 담백하게. 전문용어는 평이한 표현으로 풀어서.
- 짧고 핵심만. 긴 설명은 피한다.
- 막히면 예시를 먼저 제시한다.
- 기술적 배경 없이도 답할 수 있는 방식으로 질문한다.
- 한 번에 최대 2가지 질문만.
