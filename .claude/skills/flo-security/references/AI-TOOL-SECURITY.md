# 생성형 AI Tool 안전 사용 가이드

> **주의사항**: 생성형 AI는 사용자가 입력한 데이터를 학습하여 타인에게 답변으로 제공할 수 있으므로, 입력한 대화 내용이 외부로 유출되지 않도록 데이터 입력 및 파일 업로드 시 다음 사항을 반드시 준수해야 합니다.

## 생성형 AI 내 Prompt 사용 시 준수 사항

AI에 질문(Prompt)을 입력하거나 분석을 위해 파일을 업로드할 때는 **비식별화 처리**가 필수입니다.

### 데이터 입력 및 파일 업로드 가이드

| 분류 | 기본 지침 | 금지 예시 (X) | 올바른 예시 (O) |
|------|----------|---------------|-----------------|
| 개인정보 | 개인을 식별할 수 있는 정보 입력 금지 | 고객 성명, 연락처, 주소, 생년월일 포함 | '고객 A', '010-0000-0000' 등 마스킹 처리 |
| 비즈니스 정보 | 비공개 프로젝트 및 협력사 정보 보호 | 프로젝트 세부 일정, 협력사 실명 기재 | 'A사'로 사명 마스킹, 공개 가능한 범위만 입력 |
| IT 인프라 정보 | 시스템 접근 및 보안 관련 정보 노출 금지 | 내부 IP, URL, DB 접속 정보, 소스 코드, .env 파일 | '0.0.0.0', 'A DB' 등으로 치환 |

### 상세 로직 및 파일 업로드 시 주의사항

- **상세 설계 및 구조**: 내부 시스템의 서버 버전이나 사용 포트 번호 등 상세 구조를 기재하여 에러 원인을 조회해서는 안 됩니다. 대신 "내부 서버 접속 실패 시 점검할 일반적인 체크리스트"를 묻는 방식을 권장합니다.

- **문서 파일 업로드**: 파일 업로드 전 작성자(본인 포함) 및 구성원의 개인정보가 포함되었는지, 노출 금지된 주요 프로젝트 문서인지 확인해야 합니다.

- **소스 코드 및 로그**: 파일 전체 업로드를 금지하며, 하드코딩된 인증키나 계정 정보가 포함되었는지 확인 후 필요한 부분만 발췌하여 활용해야 합니다.

- **폴더/바이브코딩**: AI가 불러오는 폴더 내에 주요 설정 파일이 있는지 확인하고, 코드 생성 시 내부 중요 파일이 직접 업로드되지 않도록 주의해야 합니다.

## 업무 도구 연동 및 도입 시 준수 사항

업무용 도구(Slack, 메일 등)에 AI를 연동할 경우, 사용자가 직접 입력한 정보 외에도 시스템을 통해 회사 정보가 유출될 수 있습니다.

| 항목 | 기본 지침 | 금지 예시 (X) | 올바른 예시 (O) |
|------|----------|---------------|-----------------|
| 권한 관리 | AI의 접근 권한을 최소한으로 설정 | 비공개 채널 연동, 전체 대화 상시 모니터링 | 특정 채널/스레드만 접근, 키워드 호출 시에만 작동 |
| 데이터 보안 | 전송 데이터 필터링 및 범위 지정 | 메일/대화 전체 트리 전송, 원문 그대로 전송 | 정규표현식을 통한 개인정보 필터링 후 전송 |
| 도입 검토 | 보안 기능 지원 여부 사전 확인 | 데이터 학습 제외(Opt-out) 불가 도구 도입 | 학습 제외 옵션, 암호화 통신, 관리자 모니터링 지원 도구 |

## 기타 보안 사항 (공통)

가장 기본적인 보안 원칙은 공사와 사의 계정을 엄격히 분리하는 것입니다.

- **업무용-개인용 계정 분리**: 회사에서 발급한 공식 계정만을 통해 AI를 이용해야 합니다. 개인적으로 결제한 유료 계정이나 무료 버전의 AI에 업무 데이터를 입력하는 행위는 엄격히 금지됩니다.

- **접근 제어**: 허가받지 않은 사용자가 데이터를 입력하지 못하도록 사용자 계정 연동을 통해 접근을 제한해야 합니다.

## 생성형 AI 안전 사용 자가 점검 체크리스트

데이터 전송 또는 파일 업로드 전, 다음 항목을 빠짐없이 체크하십시오.

### [데이터 및 파일 내용 점검]

- [ ] 입력 정보에서 고객 및 임직원의 성명, 연락처 등 개인 식별 정보를 모두 제거하거나 마스킹했는가?
- [ ] 프로젝트 세부 일정이나 고객사/협력사 실명 등 비공개 비즈니스 정보가 포함되지 않았는가?
- [ ] 업로드 파일의 속성(메타데이터)이나 본문에 작성자 및 구성원의 개인정보가 없는가?
- [ ] 소스 코드/로그 업로드 시 주석을 포함하여 인증키, 서버 정보, 계정 정보가 제거되었는가?
- [ ] 상세 시스템 설계나 미공개 프로젝트의 구체적 타겟 전략을 그대로 기술하지 않았는가?
- [ ] 폴더 호출 시 중요 설정 파일이나 민감 코드가 .aiignore 등으로 제외되었는가?

### [업무 도구 연동 및 환경 점검]

- [ ] AI의 접근 권한이 특정 채널/스레드로 최소화되어 있으며 키워드 호출 방식으로 설정되었는가?
- [ ] 연동 대상에 비공개 채널이나 비공개 티켓 시스템이 포함되어 있지 않은가?
- [ ] API 전송 전 개인정보 필터링 절차를 거치며, 불필요한 대화 트리가 전송되지 않는가?
- [ ] 해당 AI 도구가 데이터 학습 제외(Opt-out) 옵션을 제공하며 활성화되어 있는가?

### [계정 및 서비스 설정 점검]

- [ ] 개인용 계정이 아닌 회사에서 공식 발급하고 승인한 보안 계정을 사용 중인가?
- [ ] 암호화 통신(HTTPS 등)이 적용된 안전한 경로를 통해 데이터를 주고받는가?
- [ ] 관리자가 사용 현황 및 데이터 전송 현황을 모니터링할 수 있는 도구인가?
- [ ] 내가 입력하는 정보가 유출될 경우 회사나 개인에게 피해를 입힐 가능성이 없는가?
